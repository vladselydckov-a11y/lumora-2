# v7.3.1 Clean Data Demo

Эта версия исправляет главную проблему v7.3: визуально странные суммы, когда дневные показатели смешивались с оборотом официантов за 30 дней.

## Что изменено

- В `/api/summary` добавлено разделение периодов:
  - текущий день: выручка, чеки, средний чек, фудкост, скидки, официанты, топ блюд;
  - последние 7 дней: график недели;
  - последние 30 дней: AI-анализ, слабые блюда, общий аудит.
- Вкладка `Неделя` теперь показывает подпись `Официанты за выбранный день`, а не месячный оборот команды.
- `waiters30Days` и `topDishes30Days` остаются в API для AI-аудита.
- Видимые подписи обновлены до `v7.3.1 Clean Data Demo`.

## Что делать после деплоя

Если Supabase уже заполнен старыми CSV из v7.3, заново таблицы можно не загружать. Достаточно задеплоить v7.3.1 в Vercel с теми же переменными:

```env
USE_SUPABASE=true
SUPABASE_URL=...
SUPABASE_SERVICE_ROLE_KEY=...
BOT_TOKEN=...
```

После redeploy открыть:

```text
/api/summary
```

В JSON должны быть поля `dataMode: supabase_real_clean_v7_3_1`, `dataRange`, `waiters`, `waiters30Days`, `topDishes`, `topDishes30Days`.

## Что показывать клиенту

Клиенту показываем не `/api/summary`, а обычный mini app. Объяснение простое:

```text
Мы берём выгрузку iiko/Excel, загружаем данные в систему, и mini app показывает понятную AI-аналитику: день, неделю, 30-дневный анализ, команду, блюда, риски и действия.
```

# Resto Mini App v7.4 AI Brain

Версия v7.4 усиливает AI-чат без изменения основного дизайна Dark Operator.

## Что изменилось

1. `/api/ai-chat` теперь берёт реальные данные из Supabase, если включён `USE_SUPABASE=true`.
2. AI получает не сырой огромный JSON, а чистую управленческую сводку ресторана.
3. Добавлена классификация вопроса: деньги, выручка, средний чек, фудкост, скидки, команда, меню, прогноз, отчёт владельцу.
4. Добавлен новый ресторанный system prompt: AI отвечает как операционный директор ресторана.
5. Добавлен улучшенный fallback-режим, если `OPENAI_API_KEY` ещё не подключен или OpenAI временно не ответил.
6. В интерфейсе AI-чата добавлены более сильные быстрые вопросы для демонстрации клиенту.

## Какие переменные нужны в Vercel

Обязательные для Telegram и Supabase:

```env
BOT_TOKEN=telegram_bot_token
USE_SUPABASE=true
SUPABASE_URL=https://xxxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=service_role_key
```

Для настоящего AI:

```env
OPENAI_API_KEY=sk-proj-...
OPENAI_MODEL=gpt-5.4-mini
```

Если модель `gpt-5.4-mini` недоступна в аккаунте, временно поставь модель, которая доступна в твоём OpenAI API-проекте.

## Как проверять

1. Открыть `/api/summary` и убедиться, что есть `dataMode: "supabase_real_ai_brain_v7_4"`.
2. Открыть Mini App.
3. Перейти во вкладку `AI-чат`.
4. Задать вопросы:
   - Где ресторан теряет деньги?
   - Что сделать сегодня до вечера?
   - Почему средний чек ниже цели?
   - Какие блюда стоит продвигать?
   - Кто из официантов просел?
   - Сделай короткий отчёт владельцу.

## Главное отличие

Раньше AI мог путаться, потому что получал данные слишком общо или уходил в demo/fallback-логику. В v7.4 AI работает по более жёсткому коридору:

```text
вопрос владельца → классификация → чистая сводка данных → ресторанный prompt → управленческий ответ
```
